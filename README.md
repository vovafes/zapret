## О проекте

**zapret-discord** — лёгкая утилита с открытым исходным кодом для Windows, которая восстанавливает работу Discord в условиях ограничений провайдеров, использующих системы **ТСПУ** (Технические Средства Противодействия Угрозам) / **DPI** (Deep Packet Inspection), без использования сторонних VPN-серверов и без изменения системных DNS-настроек.

Принцип работы основан на **локальной десинхронизации TCP-пакетов**: программа перехватывает исходящий TLS-хэндшейк на уровне ядра через драйвер **WinDivert** и разбивает пакет `TLS ClientHello` на несколько фрагментов так, что система DPI провайдера не успевает прочитать поле **SNI** (Server Name Indication) и применить блокировку — при этом сервер Discord корректно собирает фрагменты и устанавливает полноценное соединение.

**Главная особенность** — **OTA-обновление конфигурации**: параметры работы хранятся в файле [`config.json`](https://github.com/vovafes/zapret/blob/main/config.json) прямо в этом репозитории. Это позволяет адаптировать обход под новые методы блокировки без переустановки и пересборки программы у всех пользователей одновременно.

## ✨ Ключевые особенности

| Функция | Описание |
|---|---|
| ☁️ **Облачная конфигурация (OTA)** | При каждом запуске программа синхронизирует параметры с `vovafes/zapret` на GitHub. Автор может менять метод обхода для всех пользователей разом, обновив один файл `config.json`. |
| 🔑 **Автоповышение прав (UAC)** | При запуске без прав администратора программа автоматически показывает UAC-запрос через `ShellExecuteW`. Не нужно вручную выбирать «Запуск от имени администратора». |
| 📡 **Обход ТСПУ для текста и медиа** | Фрагментация `TLS ClientHello` обходит DPI-фильтры для Discord API, CDN и Gateway. |
| 🎙️ **Фикс голосовых и видеоканалов** | Специальный UDP-режим обфускации восстанавливает работу голосовых чатов и видеозвонков (UDP-порты 50000–65535). |
| 🚫 **Без сторонних VPN-серверов** | Весь трафик идёт напрямую к серверам Discord. Нет задержек туннеля, нет утечек данных. |
| 🛠️ **Два режима десинхронизации** | `split` (фрагменты в прямом порядке) и `disorder` (фрагменты в обратном порядке) — выбирается в `config.json`. |
| ⚡ **Минимальное влияние на производительность** | Фильтруются только пакеты на порт 443 (TCP) и порты 50000–65535 (UDP). Остальной трафик проходит без задержек. |

## Системные требования

| Компонент | Требование |
|---|---|
| **Операционная система** | Windows 10 или Windows 11 (только 64-бит) |
| **Права** | Администратор (запрашиваются автоматически) |
| **Драйвер** | `WinDivert.dll` + `WinDivert64.sys` (входят в архив релиза) |
| **Сеть** | Доступ к GitHub для синхронизации конфига (необязателен) |
| **Python** | 3.10+ (только при запуске из исходников) |

> ⚠️ Windows 7, 8, 8.1 и 32-битные версии Windows **не поддерживаются**.

## 🚀 Быстрый старт (для обычных пользователей)

> Этот способ **не требует** установки Python или каких-либо технических навыков.

### Шаг 1 — Скачайте архив

Перейдите на страницу [**Releases**](https://github.com/vovafes/zapret/releases) и скачайте последний архив `zapret-discord-vX.X.X.zip`.

### Шаг 2 — Распакуйте в любую папку

После распаковки убедитесь, что в папке присутствуют **все три файла**:

```
📁 zapret-discord\
  ├── zapret-discord.exe    ← основной исполняемый файл
  ├── WinDivert.dll         ← обязательно!
  └── WinDivert64.sys       ← обязательно!
```

> ⛔ **Критически важно:** `WinDivert.dll` и `WinDivert64.sys` **обязаны** находиться в той же папке, что и `zapret-discord.exe`. Без них программа не запустится.

### Шаг 3 — Запустите

Дважды кликните по `zapret-discord.exe`. Появится системный запрос **Контроля учётных записей (UAC)** — нажмите **«Да»**.

Программа автоматически:
1. Запросит права администратора
2. Загрузит актуальный конфиг с репозитория `vovafes/zapret`
3. Активирует перехват трафика Discord

**Оставьте окно программы открытым.** Для корректной остановки нажмите **Ctrl+C**.

## ⚙️ Конфигурация (`config.json`)

При каждом запуске программа загружает конфигурацию с:

```
https://raw.githubusercontent.com/vovafes/zapret/main/config.json
```

Если GitHub недоступен, программа использует встроенную конфигурацию по умолчанию.

### Структура

```json
{
  "target_domains": [
    "discord.com",
    "discordapp.com",
    "gateway.discord.gg",
    "discord.media",
    "cdn.discordapp.com"
  ],
  "split_position": 2,
  "desync_mode": "split",
  "udp_fake_enabled": true
}
```

| Параметр | Тип | По умолчанию | Описание |
|---|---|---|---|
| `target_domains` | `string[]` | — | Домены Discord для перехвата |
| `split_position` | `integer` | `2` | Позиция разбивки TLS ClientHello (байт) |
| `desync_mode` | `string` | `"split"` | `"split"` или `"disorder"` |
| `udp_fake_enabled` | `boolean` | `true` | UDP-обфускация для голосовых каналов |

## 🔬 Как это работает (Архитектура)

```
 ┌──────────────────────────────────────────────────────────────────┐
 │                Discord App (браузер / клиент)                    │
 │    Исходящий пакет: TCP → TLS ClientHello (SNI: discord.com)     │
 └────────────────────────────┬─────────────────────────────────────┘
                              │  (пакет перехвачен до отправки)
                              ▼
 ┌──────────────────────────────────────────────────────────────────┐
 │                     zapret-discord                               │
 │              (WinDivert — уровень ядра Windows)                  │
 │                                                                  │
 │  1. Перехват исходящего TCP-пакета на порт 443                   │
 │  2. Обнаружение TLS Handshake: byte[0]=0x16, byte[5]=0x01        │
 │  3. Поиск SNI в payload: "discord.com" найден ✓                  │
 │  4. Фрагментация payload на две части                            │
 │  5. Отправка фрагментов с корректными TCP SEQ-номерами           │
 └───────────────┬────────────────────────┬────────────────────────┘
                 │ Фрагмент ①             │ Фрагмент ②
                 ▼                        ▼
 ┌──────────────────────────────────────────────────────────────────┐
 │                  ТСПУ / DPI провайдера                           │
 │  Видит неполный ClientHello — SNI не определяется                │
 │  Правило блокировки не срабатывает → оба фрагмента пропущены ✓   │
 └───────────────────────────────┬──────────────────────────────────┘
                                 ▼
 ┌──────────────────────────────────────────────────────────────────┐
 │                    Сервер Discord                                │
 │  TCP-стек собирает фрагменты по SEQ-номерам                      │
 │  Полный ClientHello → TLS Handshake завершён → соединение ✓      │
 └──────────────────────────────────────────────────────────────────┘
```

### Детали реализации

**WinDivert** встраивается в сетевой стек Windows (WFP/NDIS) и перехватывает пакеты до их передачи адаптеру. Это локальная операция — трафик не покидает компьютер через посредника.

**TLS ClientHello** определяется по сигнатуре: `byte[0] == 0x16` (Handshake) и `byte[5] == 0x01` (ClientHello). Расширение SNI с именем домена содержится в пакете в открытом виде.

**Режим Split:** payload разбивается на две части в позиции `split_position`. DPI анализирует каждый сегмент независимо и не видит полного SNI в первом из них.

**Режим Disorder:** второй фрагмент отправляется первым. Большинство DPI-реализаций не выполняют переупорядочивание TCP-сегментов.

**UDP-обфускация:** перед каждым голосовым пакетом отправляется короткий фиктивный пакет, сбрасывающий состояние UDP-трекера DPI.

**OTA-конфигурация:** параметры хранятся в `config.json` в репозитории `vovafes/zapret`. При изменении методов блокировки достаточно обновить этот файл — все пользователи получат новые настройки при следующем запуске.

## 🛡️ Антивирусы и ложные срабатывания

Программы, работающие с сетевым трафиком на уровне ядра через WinDivert, могут вызывать ложные срабатывания антивирусного ПО. Это ожидаемое поведение — программа не содержит вредоносного кода.

Если антивирус блокирует запуск, добавьте папку с программой в исключения.

## ❓ Частые вопросы

**Q: Discord всё равно не подключается.**  
A: Попробуйте сменить `"desync_mode"` с `"split"` на `"disorder"` в `config.json`. Также попробуйте увеличить `"split_position"` до 4 или 6.

**Q: Голосовые каналы не работают.**  
A: Убедитесь, что `"udp_fake_enabled": true`. В настройках Discord попробуйте сменить голосовой регион сервера.

**Q: Программа закрылась — интернет работает нормально?**  
A: Да. При нажатии Ctrl+C программа корректно закрывает хэндл WinDivert и весь трафик снова идёт напрямую без обработки.

**Q: Работает ли на Windows 7 / 32-бит?**  
A: Нет. Только Windows 10 и 11 x64.
